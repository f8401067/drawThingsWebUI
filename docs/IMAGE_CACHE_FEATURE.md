# 浏览器图片缓存功能

## 功能说明

本项目已实现基于 Service Worker 的浏览器端图片缓存功能，可以显著提升图片加载速度并减少服务器带宽消耗。

### 核心特性

1. **自动缓存**：当用户访问任何图片时，Service Worker 会自动将其缓存到浏览器中
2. **智能策略**：采用 "stale-while-revalidate" 策略
   - 优先从缓存加载（快速响应）
   - 后台静默更新缓存（保持内容最新）
3. **容量管理**：最多缓存 50 张图片，超出时自动删除最旧的图片
4. **离线支持**：即使网络断开，已缓存的图片仍可正常访问
5. **默认显示全部**：历史页面默认不设置日期筛选，显示所有日期的图片，充分利用缓存

## 缓存特性

- ✅ **自动生效**：无需任何配置，首次访问后自动开始缓存
- ✅ **透明运行**：用户无感知，不影响现有功能
- ✅ **性能优化**：第二次访问同一图片时几乎瞬间加载
- ✅ **空间管理**：自动清理旧缓存，避免占用过多存储空间
- ✅ **跨页面共享**：首页和历史页共享同一缓存

## 使用方法

### 查看缓存状态

1. 打开历史图片页面 (`/history.html`)
2. 展开筛选条件面板
3. 点击 **"查看缓存"** 按钮
4. 将显示当前缓存的图片数量和列表

### 清除缓存

1. 打开历史图片页面 (`/history.html`)
2. 展开筛选条件面板
3. 点击 **"清除缓存"** 按钮
4. 确认后，所有缓存将被清空，页面会自动刷新

## 技术细节

### 缓存位置
- 浏览器 Cache Storage API
- 缓存名称：`drawthings-images-v1`
- 最大容量：50 张图片

### 缓存的文件类型
- PNG (.png)
- JPEG/JPG (.jpg, .jpeg)
- GIF (.gif)
- WebP (.webp)

### 缓存策略
```
请求图片
  ↓
检查缓存
  ├─ 命中 → 立即返回缓存 + 后台更新
  └─ 未命中 → 从服务器获取 → 存入缓存 → 返回
```

### 手动查询缓存（控制台）

在浏览器控制台中执行：

```javascript
// 获取缓存统计
caches.open('drawthings-images-v1').then(async cache => {
  const keys = await cache.keys();
  console.log('Cached images:', keys.length);
  keys.forEach(k => console.log(k.url));
});

// 清除所有缓存
caches.delete('drawthings-images-v1');
```

### 调试技巧

1. **查看 Service Worker 状态**
   - Chrome: `chrome://serviceworker-internals/`
   - Firefox: `about:debugging#/runtime/this-firefox`

2. **强制更新 Service Worker**
   ```javascript
   // 在控制台中执行
   navigator.serviceWorker.getRegistrations().then(registrations => {
     registrations.forEach(reg => reg.unregister());
   });
   ```

3. **查看缓存日志**
   - 打开浏览器控制台
   - 筛选 `[SW]` 标签
   - 可以看到缓存命中/未命中的详细信息

## 注意事项

⚠️ **重要提示**：

1. **HTTPS 要求**：Service Worker 只能在 HTTPS 或 localhost 环境下工作
2. **浏览器支持**：需要现代浏览器（Chrome 40+, Firefox 44+, Safari 11.1+）
3. **存储空间**：浏览器可能根据磁盘空间限制自动清理缓存
4. **首次加载**：第一次访问图片时仍需要从服务器加载，之后才会使用缓存

## 故障排除

### Service Worker 未注册

**症状**：控制台显示 "Registration failed"

**解决**：
1. 确保使用 HTTPS 或 localhost
2. 检查浏览器是否支持 Service Worker
3. 清除浏览器缓存后重试

### 缓存不生效

**症状**：每次加载图片都很慢

**解决**：
1. 打开控制台查看 `[SW]` 日志
2. 确认图片 URL 格式正确（以 .png/.jpg 等结尾）
3. 检查浏览器存储设置是否禁用了缓存

### 缓存占用过多空间

**解决**：
1. 使用 "清除缓存" 按钮
2. 或在控制台执行：`caches.delete('drawthings-images-v1')`
